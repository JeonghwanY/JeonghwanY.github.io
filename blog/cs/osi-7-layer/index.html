OSI 7계층(OSI 7 Layer): 네트워크 통신을 7단계로 나눈 설계도

04 May 2025 in Blog / CS on 네트워크
  • 1. 한눈에 보는 OSI 7계층
  • 2. 계층별 세부사항
    • 1계층. 물리 (Physical)
    • 2계층. 데이터링크 (Data-Link)
    • 3계층. 네트워크 (Network)
    • 4계층. 전송 (Transport)

OSI 7계층은 물리 신호부터 사용자 애플리케이션까지, 네트워크 기능을 7단계로 나눈 개념적 설계도다. 실무에서는 TCP/IP 4계층으로 단순화해 쓰지만, 문제 분석과 개념 정리에는 여전히 유용하다. 각 계층마다 맡은 역할이 정해져 있어, 복잡한 통신 과정을 구조화하고 표준화하는 데 도움을 준다.

1. 한눈에 보는 OSI 7계층

번호 계층 이름 주요 기능 예시
7 응용 (Application) 사용자와 직접 상호작용, 네트워크 서비스 제공 HTTP/HTTPS, FTP, SMTP, DNS, SSH
6 표현 (Presentation) 데이터 형식 변환, 암호화/복호화, 압축 JPEG, MP3, SSL
5 세션 (Session) 연결 설정·유지·종료, 세션 관리 NetBIOS, RPC, PPTP
4 전송 (Transport) 데이터 전송 신뢰성 보장, 오류 제어 TCP, UDP, QUIC, SCTP
3 네트워크 (Network) 주소 지정, 경로 선택(라우팅) IP, ICMP, ARP, OSPF, BGP
2 데이터링크 (Data-Link) 프레임 생성, MAC 주소 기반 전달, 에러 감지 Ethernet, Wi-Fi(802.11), PPP, VLAN, 스위치
1 물리 (Physical) 0/1 신호를 실제 매체로 전송 UTP 케이블, 광섬유, 전파, 허브, 리피터

2. 계층별 세부사항

물리 계층(1)부터 응용 계층(7)까지, 아래에서 위로 올라가며 추상화 수준이 높아진다.

1계층. 물리 (Physical)

“전깃줄에 0과 1을 어떻게 흘려보낼까?”

  • 전압 크기, 광펄스 세기, 라디오 주파수 등 신호의 물리적 특성을 정의한다
  • 허브·리피터처럼 “증폭만 하고 아무 판단도 못 하는” 장비가 여기에 속한다

2계층. 데이터링크 (Data-Link)

“같은 네트워크(브로드캐스트 도메인) 안에서 누가 누구인지 구분하려면?”

  • MAC 주소를 붙여 프레임 단위로 전송하며, CRC로 프레임 오류를 검출한다
  • 스위치·브리지, VLAN, ARP 캐시 같은 개념이 등장한다

3계층. 네트워크 (Network)

“지구 반대편 호스트까지 가려면 중간 경로(라우팅)를 어떻게 잡을까?”

  • IP 주소를 붙이고, 라우터가 최적 경로를 찾아 포워딩한다
  • ICMP의 ‘목적지 도달 불가’, OSPF/BGP 같은 라우팅 프로토콜이 여기에 있다

4계층. 전송 (Transport)

“프로세스 간에 패킷이 유실돼도 재전송하고, 순서도 맞춰야지”

  • TCP: 3-Way Handshake, 흐름/혼잡 제어, 세그먼트 재전송
  • UDP: 최소한의 헤더로 “빠르지만 책임지지 않는” 전송
  • QUIC: HTTP/3가 쓰는 신흥 강자. UDP 위에 신뢰성과 TLS를 포함

5계층. 세션 (Session)

“논리적인 대화방(세션)을 열고 닫는 관리를 맡자”

  • 체크포인트·재동기화·다중 스트림 제어가 가능하다
  • 실무에서는 구분이 애매하다. OSI만의 전통적 구분이며, 실제 구현은 주로 응용/전송 계층이 흡수한다

6계층. 표현 (Presentation)

“바이트를 사람이 읽을 수 있게, 또는 암호화된 형태로 변환하자”

  • 문자 집합 변환(UTF-8 ↔ EUC-KR), 압축(GZIP), 암복호화(TLS)를 담당한다
  • 응용 계층과 합쳐 다루는 경우가 많지만, 보안·멀티미디어 코덱을 설명할 때 유용하다

7계층. 응용 (Application)

“사용자 프로그램이 원하는 서비스 그 자체”

  • 웹브라우저-HTTP, 메일-SMTP/IMAP, 파일전송-FTP/SFTP처럼 실제 앱 프로토콜이 자리한다
  • API 호출(REST/GraphQL), 메신저 프로토콜도 여기서 논의된다

3. 왜 TCP/IP 4계층을 더 많이 쓸까?

OSI 계층 TCP/IP (실무) 설명
7–5 응용 계층 HTTP·TLS·RPC 등을 한 덩어리로 묶음
4 전송 계층 TCP·UDP·QUIC
3 인터넷 계층 IP, ICMP 등
2–1 네트워크 접근 계층 Ethernet·Wi-Fi·물리 매체
  • 간소화: 실제 구현에서 세션·표현 계층이 따로 분리돼 작동하지 않는 경우가 많아 4단계로 줄였다
  • 표준화 주체 차이: OSI는 ISO(국제표준화기구), TCP/IP는 IETF(인터넷 엔지니어링 태스크포스)가 주도한다

4. OSI 7계층을 배우는 이유

이유 설명
문제 진단 “이건 3계층 문제인가, 4계층 문제인가?”를 구분해야 디버깅이 쉽다
역할 분리 새 프로토콜 설계 시 “이 기능은 어느 계층 책임인가?”를 결정하는 기준이 된다
학습 프레임 물리 → 응용으로 올라가며 추상화와 책임 범위를 체계적으로 이해할 수 있다
